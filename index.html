Project Manager Summary

This progress update documents the current direction of the Classy Clay's Catering prototype for Milestone 2. The project is focused on creating a simple no-code/low-code system that supports menu management, catering and meal prep order intake, AI-assisted estimate generation, draft invoice/email creation, manual review, and custom order handling.

The main working prototype is being built in Airtable. Airtable stores the customer data, menu items, request/order records, AI draft records, and workflow statuses. GitHub Pages is being used as the public progress and documentation page for the project. The prototype is designed to support existing systems like Owner.com and Square instead of replacing them.

Order Intake Progress

The order intake process is being designed to collect more complete information up front. This should reduce repeated follow-up and help the owner identify whether a request is standard, custom, or missing important details.

Core Workflow Process

The Milestone 2 prototype follows a seven-step workflow from order intake to owner review.

  1. Capture the order: The customer submits a catering or meal prep request with customer information, event type, event date, guest count, menu items, allergies, and special requests.
  2. Classify the order: The system identifies whether the request is a standard order, missing information, or a custom order.
  3. Route the order: Standard orders continue to AI-assisted estimate generation. Custom or unclear orders are routed to manual review.
  4. Build the AI prompt: The system combines the order details with menu items and prices.
  5. Generate the estimate: AI creates a draft estimate with line items, quantities, prices, subtotal, and total.
  6. Create a draft invoice or email: The system prepares a customer-facing draft message based on the estimate.
  7. Manual review: The owner reviews, edits, approves, or rejects the draft before anything is sent to the customer.

AI-Assisted Estimate Generation

For standard catering or meal prep requests, the system is designed to use AI to help generate an estimated cost breakdown. The AI receives the customer request details and the available menu data, then returns a structured draft estimate.

Data Sent to AI

  • Customer request type
  • Event type
  • Event date
  • Guest count
  • Selected menu items
  • Allergies or dietary restrictions
  • Special requests or customer notes
  • Menu item prices and serving information

Expected AI Output

  • Itemized line items
  • Recommended quantities
  • Unit prices
  • Line totals
  • Subtotal
  • Service fee or delivery note, if applicable
  • Grand total estimate

Draft Invoice or Email Generation

After an AI-assisted estimate is created, the prototype prepares a draft customer email or invoice summary. The draft includes the customer greeting, event details, selected menu items, estimated price breakdown, and next steps.

The draft is not automatically sent to the customer. It must first be reviewed by the owner. This protects the business from sending incorrect pricing, wrong menu details, or unclear wording.

Sample Draft Email Purpose

The draft email helps Classy Clay’s respond faster while still allowing the owner to make the final decision. The owner can edit quantities, pricing, wording, or follow-up questions before approving the message.

Manual Review Before Sending

Manual review is a required part of the workflow. The prototype is designed so that no invoice, estimate, or customer email is sent automatically. The owner must review and approve the draft first.

Owner Review Options

  • Approve the draft
  • Edit the draft
  • Request more information from the customer
  • Flag the order for custom/manual review
  • Reject the draft if the estimate is not accurate

Custom and Manual Review Handling

Not every request should receive an AI estimate. Custom, unclear, or off-menu orders should be routed to the owner for manual review. This is important because some catering requests require judgment, direct communication, or special pricing.

Orders That Should Be Flagged

  • Custom menu requests
  • Off-menu items
  • Missing guest count
  • Missing menu item selections
  • Unusual dietary restrictions
  • Large or complex catering events
  • Requests that need direct owner approval

Testing Evidence

The prototype should be tested with at least two realistic scenarios: one standard order and one custom or unclear order.

Standard Order Test

A standard order should include a clear event type, guest count, selected Classy Clay’s menu items, and no unusual special requests. The expected result is that the system generates a draft estimate and draft email for owner review.

Custom Order Test

A custom order should include an off-menu request, vague order details, or a custom menu request. The expected result is that the system skips the automated AI estimate and routes the order to manual review.

Known Limitations

The current prototype is still a class project prototype. Some features are working in Airtable, while other parts are still simulated or being tested.

  • AI estimate generation still needs additional testing for accuracy.
  • Email sending is not fully live yet and may be represented as a draft workflow.
  • Make.com automation testing is still in progress.
  • Custom orders require manual review and should not be automatically estimated.
  • Owner.com and Square are not fully replaced by this prototype.
  • Final pricing still requires owner review before being sent to a customer.
