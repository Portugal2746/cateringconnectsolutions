## Purpose
This repository supports the project progress update for Classy Clay's Catering. It documents the current prototype direction, team responsibilities, data model, menu management progress, order intake progress, blockers, and next steps.

## Prototype Direction
The team is building a no-code/low-code prototype that supports:

- Customer request tracking
- Catering and meal prep intake
- Menu management
- AI-assisted follow-up drafts
- Marketing content support
- Repeat customer engagement
- Dashboard and status tracking

The system is designed to complement existing tools such as Owner.com and Square. It does not replace Square's estimate, invoice, or payment workflow.

### Current Prototype Status

The main working prototype is being built in Airtable. Airtable stores the customer information, menu items, catering and meal prep requests, AI draft records, and marketing/campaign records. The GitHub Pages site is used as a public project progress page that summarizes the team’s work and links to the prototype resources.

### Milestone 2 Workflow

The prototype is designed to show the following workflow:

1. Menu data is stored in the system.
2. A catering or meal prep order is created.
3. The system reviews order details and identifies missing information.
4. Standard orders move forward for AI-assisted estimate generation.
5. Custom or unclear orders are flagged for manual review.
6. A draft invoice or customer email is created.
7. The owner reviews the draft before anything is sent to the customer.

### Known Limitations

The current prototype is still a class project prototype. Some automation steps may be simulated, and live customer email sending is not fully enabled yet. Owner review is required before any message or invoice is sent. Additional testing is still needed for custom orders, menu updates, and AI estimate accuracy.
